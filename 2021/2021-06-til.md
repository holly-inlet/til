    - ドキュメントファイル内に`docker-build`ファイル作成，その中にDockerfile作成。
    - ブログのubuntu，Anaconda3インストールコードをDockerfileにコピペ。
    - `$ docker build .`でビルド
      - 実行結果に`writing image sha256:~`と表示され，`$ docker images`をやっても新しいimageが見当たらない。
      - `sha256`の意味を調べるがわからない。
      - 再度`$ docker images`を実行すると，`REPOSITORY`と`TAG`がnoneのイメージがあった。
      - 結局，見落としていただけで，何の問題も起きていなかった。
    - BuildしたイメージをRunする
      - `$ docker run -p 8888:8888 -v ~/Documents/Programming/Python:/work --name my-lab 764088c09067`を実行。
      - コンテナのポートをホストのポートにマッピング  
      `-p {host port}:{container port}`
      - ホストとコンテナのファイルシステムのマウント  
      `-v {host volume}:{container volume}`
      - コンテナの名前を指定  
      `--name {コンテナ名}`
      - ブラウザから`localhost:8888`のページに飛ぶ  
        → 無事Jupyter Labが表示され，コードを書く環境が整った。
  - 次は[データサイエンスのためのPython入門②〜Jupyter Labの使い方とショートカット一覧まとめ〜——米国データサイエンティストのブログ](https://datawokagaku.com/howtouse_jupyter/)を見ながらJupyter Labの使い方を覚えることにする。
